Running PCA on Wildfire Data

As part of a project to predict wildfires I partook in recently I was tasked with performing some basic data analysis on a csv file compiled from scraped weather data. This post details how I took the csv and performed Principal Component Analysis on it.

The first step was to import necessary files. In this case I needed an efficient way to perform PCA and plot the resulting graph. SciKit-Learn and Matplotlib both lend themselves well to this. In addition pandas and numpy are both great resources for data manipulation so these were imported as well.

%matplotlib inline

import matplotlib
import pandas as pd
import numpy as np
from sklearn import preprocessing
from sklearn.decomposition import PCA
from sklearn.model_selection import train_test_split
import matplotlib.pyplot as plt

Next the data was loaded into a pandas dataframe and split it into train/test samples. Whilst not essential for PCA this splitting was applied for the purpose of training and testing models later on. I will perhaps write another post detailing the structure of Balanced.csv. For the purpose of understanding this post all the reader needs to know is that it contains 8 meaningful features such as avg Temperature, humidity etc. The final column contains boolean values giving whether or not there was a wildfire on a day corresponding to the values of the 8 features. These were treated as class labels. 0 for no wildfire and 1 for a wildfire.

InÂ [4]:
#load in data
data = pd.io.parsers.read_csv(
    'Data/Balanced.csv',
    header=None,
    skiprows = [0],
    usecols=[5,10,15,16,17,18,19,20,22]
    )



X = data.values[:,:8]

y = data.values[:,8]

#split the data into training and testing data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.30, random_state=12345)

The data was then standardised. Most models do not perform well on non-standardised data and since it was intended to train and test a model on this data later on it was important to standardise now. It is important to standardise both train and test data under the same scale to avoid inaccurate scores during the testing phase.

#standardisation using sklearn
std_scale = preprocessing.StandardScaler().fit(X_train)
X_train_std = std_scale.transform(X_train)
X_test_std = std_scale.transform(X_test)

At this point PCA was applied to the training data:

InÂ [6]:
#Perform PCA on standardised variables
pca_std = PCA(n_components=2).fit(X_train_std)
X_train_std = pca_std.transform(X_train_std)

I opted to generate two eigenvectors since the aim in applying PCA was to try and catch a general trend in the data. Lets see how this looks.

fig, ax2 = plt.subplots(figsize=(10,4))


for l,c,m in zip(np.unique(y_train), ('blue', 'green', 'red'), ('^', 's', 'o')):
    ax2.scatter(X_train_std[y_train==l, 0], X_train_std[y_train==l, 1],
    color=c,
    label='class %s' %l,
    alpha=0.5,
    marker=m
    )
   
ax2.set_title('Transformed standardized training dataset after PCA')    


ax2.set_xlabel('1st principal component')
ax2.set_ylabel('2nd principal component')
ax2.legend(loc='upper right')
ax2.grid()
plt.tight_layout()

plt.show()

Remembering that class 1 is given by wildfires and class 0 no wildfires we can see that as we move down the y-axis wildfires begin to cluster together. This is promising as it appears in the very least that wildfires dont occur randomly (as one would expect!) However it's not all good news. The dataset seems to contain many anomalous values. Clearly these values would have a negative impact on the accuracy of any model so developing a method of reducing these anomalous values became the next focus.
